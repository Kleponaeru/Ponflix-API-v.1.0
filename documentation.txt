# Anime API Project Context

## Goal

Build a clean, provider-based Anime REST API using **Next.js 16 App Router + TypeScript**.

The API should expose a consistent JSON format while scraping data from anime providers. We originally started with Kuramanime but switched to **Kuronime** because Kuramanime blocks production/Vercel requests (403 Forbidden).

---

# Current Provider

Current provider:

* Kuronime
* Base URL:

  ```ts
  export const BASE_URL = "https://kuronime.sbs";
  ```

All scraping currently targets Kuronime.

# Architecture

The project follows a strict separation of responsibilities.

## client.ts

Responsible only for HTTP requests.

Example:

```ts
const html = await kuronimeClient.get("/");
```

No parsing logic belongs here.

---

## latest.ts / search.ts

Responsible only for:

1. Fetch HTML
2. Pass HTML into parser

Example:

```ts
const html = await kuronimeClient.get(...);

return parseLatest(html);
```

No scraping logic should exist here.

---

## parser/

Responsible only for converting HTML into typed objects.

Example:

```
HTML
 ↓
Cheerio
 ↓
LatestAnime[]
```

Parsers never perform HTTP requests.

---

# Shared Card Parser

A reusable parser exists:

```
parser/card.ts
```

It extracts fields shared by multiple pages:

* slug
* url
* path
* thumbnail

It intentionally does **NOT** parse:

* title
* episode
* score
* views

Those are page-specific and handled inside each parser.

# Latest Endpoint

Endpoint:

```
GET /api/latest
```

Scrapes:

```
https://kuronime.sbs/
```

Parses:

* title
* slug
* thumbnail
* url
* path
* currentEpisode
* totalEpisodes (currently null)
* type (currently null)
* quality (currently null)
* hot (currently false)
* views
* timeAgo

Uses:

```
parseLatest()
```

---

# Search Endpoint

Endpoint:

```
GET /api/search?q=naruto
```

Scrapes:

```
https://kuronime.sbs/?s=<query>
```

Parses:

* title
* slug
* thumbnail
* url
* path
* type
* score

Uses:

```
parseSearch()
```

# Coding Style

Important project rules:

* Keep functions small.
* No duplicated parsing.
* Reuse parser/card.ts whenever possible.
* Services never parse HTML.
* Parsers never perform HTTP requests.
* Prefer readable code over clever code.
* Fix architecture before adding features.
* Avoid temporary hacks that require later refactoring.
* Make changes cleanly the first time.

# CURRENT ISSUES (Kuronime Provider - Episode Streaming Focus)

## 1. No Valid Streaming Server Extraction
- decryptSource() does not produce usable server list data
- servers array always empty (`[]`)
- no playable video URLs are being returned

## 2. Encrypted Source Not Converted to Server Format
- API returns encrypted fields (src / src_sd / mirror)
- these are never successfully transformed into:
  - embed servers
  - download servers
  - playable links

## 3. Missing Server Schema Mapping
- decrypted output (when it exists) is not consistently mapped to EpisodeServer[]
- normalizeServers() receives invalid or empty structured input

## 4. No Verified Stream Endpoint Output
- no confirmed final output like:
  - iframe URL
  - direct video URL
  - provider embed link
- episode response does not contain real playback sources

## 5. Mirror / src_sd / src Not Utilized Properly
- multiple potential sources exist:
  - src
  - src_sd
  - mirror
- but none are reliably decoded into server entries

## 6. Fallback Server Logic Not Functional
- Filelions fallback only works if decrypt succeeds
- blog fallback exists but not tied to actual streaming flow
- no guaranteed server output path

## 7. Normalization Depends on Unreliable Input
- normalizeServers() assumes structured decrypted object
- current decrypted output is inconsistent or null
- leads to empty server list every time
